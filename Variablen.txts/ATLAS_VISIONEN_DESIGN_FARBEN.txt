3. TYPOGRAPHY — SCHRIFT & HIERARCHIE
================================================================================

FONT-EMPFEHLUNG:
  Primary:    Inter          → universell, clean, developer-fokussiert
  Alternative: Geist         → von Vercel, sehr modern
  Alternative: Satoshi       → etwas organischer, trotzdem tech
  Monospace:  JetBrains Mono → Code, Terminal, Daten

FONT WEIGHTS:
  Nur zwei Gewichte: 400 (regular) und 700 (bold)
  Kein 500, kein 600 — wirkt zu mittelmäßig
  Kontrast zwischen Headlines (bold) und Body (regular) ist der Trick

TYPOGRAFIE-HIERARCHIE:

  Hero Headline:    64–80px  font-weight: 700  letter-spacing: -0.02em  line-height: 1.1
  Section Headline: 40–56px  font-weight: 700  letter-spacing: -0.01em  line-height: 1.2
  Subheadline:      24–32px  font-weight: 400  letter-spacing: 0        line-height: 1.4
  Body Text:        16px     font-weight: 400  letter-spacing: 0        line-height: 1.6
  Label / Tag:      12–13px  font-weight: 500  letter-spacing: 0.05em   UPPERCASE
  Code / Mono:      14px     JetBrains Mono    letter-spacing: 0

TYPOGRAFIE-PRINZIPIEN:
  "Huge headline, short subline, minimal body text"
  Sehr viel Whitespace um Headlines
  Text kommuniziert Vision, nicht Features
  Wenige Wörter pro Section (je weniger, desto premium)
  Kein Dense Text — niemals lange Absätze in Hero-Bereich

HERO-BEISPIELE (aus dem Dokument):
  "Navigate the AI Universe."
  "The Navigation Layer of Artificial Intelligence."
  "Every model. Every tool. One atlas."
  "AI is exploding. But no one knows where anything is."

PSYCHOLOGIE:
  Große Headlines = Größe kommunizieren
  Die Seite verkauft Vision durch Typografie, nicht durch Text-Menge

5. DESIGN PRINZIPIEN — 20 KERN-REGELN
================================================================================

PRINZIP 01 — ENVIRONMENT > INTERFACE
  Die Seite ist kein UI. Die Seite ist ein Raum.
  Nutzer betritt eine Umgebung, keine Website.

PRINZIP 02 — SCALE > INFORMATION
  80% Raum, 20% Content.
  Objekte sind klein, Umgebung ist riesig.
  Das erzeugt: "Dieses System ist größer als ich."

PRINZIP 03 — NARRATIVE > NAVIGATION
  Scrollen = Story erleben, nicht navigieren.
  Jede Section = Kapitel, nicht Feature-List.

PRINZIP 04 — HERO = 1 MESSAGE
  Eine Headline. Ein Satz. Eine CTA.
  Kein Bullet-Point-Hero. Keine Feature-Aufzählung.

PRINZIP 05 — MAXIMAL 1 PRIMÄRER CTA PRO SECTION
  Klarheit durch Fokus.
  Mehr Optionen = mehr Unsicherheit = weniger Klicks.

PRINZIP 06 — DARK UI = DEVELOPER SIGNAL
  Dark Interfaces kommunizieren:
  Präzision, Kontrolle, technische Kompetenz.
  Vercel, Linear, GitHub, Raycast — alle dark.

PRINZIP 07 — NEGATIVE SPACE = PREMIUM SIGNAL
  Apple, Stripe, Linear — alle nutzen massiv Leerraum.
  Volle Seiten wirken billig. Leere Seiten wirken teuer.

PRINZIP 08 — SECTIONS = EVOLUTION, NICHT FEATURES
  Nicht: "Feature A, Feature B, Feature C"
  Sondern: "Tool → Copilot → Agent → Ecosystem"
  User versteht Wachstum, nicht Liste.

PRINZIP 09 — PRODUKT VISUELL ERKLÄREN
  Screenshots, UI-Previews, Galaxy Map als Hero Visual
  User versteht Produkt durch Bilder, nicht Text.

PRINZIP 10 — MOTION SPEED = WERTIGKEIT
  Langsame Animationen = Premium, teuer
  Schnelle Animationen = billig, busy
  Cinematic Motion: 600–1200ms transitions, ease-in-out

PRINZIP 11 — FARBE KODIERT BEDEUTUNG
  Farbe identifiziert Kategorien (nicht Dekoration)
  Konsistenz: Image Gen immer blau, LLMs immer purple

PRINZIP 12 — TIEFENILLUSION = IMMERSION
  Background + Midground + Foreground
  Parallax erzeugt 3D-Raumtiefe ohne echtes 3D
  Stars (langsam) → Nebula (mittel) → UI (schnell)

PRINZIP 13 — LICHT AUS DER MITTE
  Eye Attraction Point = Bildmitte
  Glow, Nodes, Headlines → immer mittig
  Blick wird automatisch dorthin gezogen

PRINZIP 14 — WORLD FIRST
  Universe → Immersion → Information
  Nicht: Interface → Content
  User fühlt Welt zuerst, liest danach

PRINZIP 15 — MINIMAL NAVIGATION
  Nur: Product | Atlas | Tools | Docs | Get Started
  Kein Mega-Menu. Keine komplexen Dropdowns.
  Simplicität = Confidence

PRINZIP 16 — AGENT TRANSPARENCY
  Zeige: Plan → Execution → Output
  Macht AI-Prozesse sichtbar und vertrauenswürdig

PRINZIP 17 — GRAIN / NOISE OVERLAY
  Film Grain CSS Overlay (opacity: 0.04–0.06)
  Erzeugt sofort: Film Look, Depth, Premium Feeling
  Einer der günstigsten Tricks mit größtem visuellen Effekt

PRINZIP 18 — STARS = AI TOOLS
  Visuelle Metapher: AI Modelle sind Sternbilder
  GPT Constellation, Image Gen Constellation
  Atlas = Sternenkarte des AI-Universums

PRINZIP 19 — KONSTELLATIONEN = KATEGORIEN
  Cluster von Tools = Sternbilder
  Verbindungslinien = Workflows / Routen
  Zoom Navigation = Reise durch das Universum

PRINZIP 20 — MASTER FORMEL
  Environment + Extreme Scale + Cinematic Motion
  + Narrative Scroll + Network Visualization
  = Infrastructure Perception
  → Nutzer denkt: "Atlas ist der Layer über der AI-Industrie"

7. DIE 7 TEUERSTEN DESIGN-TRICKS (die eine Seite wie $500M wirken lassen)
================================================================================

TRICK 1 — NOISE / GRAIN OVERLAY
  Was: Ein Film-Grain CSS Overlay über die gesamte Seite
  Code:
    body::after {
      content: "";
      position: fixed;
      top: 0; left: 0;
      width: 100%; height: 100%;
      background: url(noise.png);
      opacity: 0.05;
      pointer-events: none;
    }
  Effekt: Sofort Film Look, Tiefe, Premium Feeling
  Kosten: 0. Eine CSS-Zeile.

TRICK 2 — SEHR LANGSAME ANIMATIONEN
  Was: Alle Transitions auf 600–1200ms setzen (statt Standard 200ms)
  Effekt: Seite fühlt sich teuer und kalkuliert an
  Formel: Duration = Budget-Signal. Langsam = teuer.

TRICK 3 — MASSIVER LEERRAUM (EMPTY SPACE)
  Was: 70–85% der Fläche ist leer
  Objekte klein, Umgebung groß
  Effekt: Luxus-Signal. Apple, Linear, Stripe machen genau das.

TRICK 4 — LICHT AUS DEM ZENTRUM
  Was: Glow-Effekte immer aus der Bildmitte
  Star-Nodes pulsieren von innen
  Effekt: Eye Attraction Point, Tiefe, natürlicher Fokus

TRICK 5 — DARK + ONE ACCENT GLOW
  Was: 99% Dark, 1% electric blue oder purple glow
  Nur interaktive Elemente bekommen die Accent-Farbe
  Effekt: Extrem fokussiert. Jedes leuchtende Element wirkt wichtig.

TRICK 6 — HERO OHNE SCREENSHOT
  Was: Hero zeigt keine UI-Screenshots sondern abstraktes System-Visual
  AI Galaxy Map / Konstellations-Netzwerk / Node-Graph
  Effekt: Produkt wirkt wie Infrastruktur, nicht wie Tool

TRICK 7 — PARALLAX MIT NUR 3 LAYERN
  Was: Background (stars/nebula, 0.2x speed)
       Midground (particles/lines, 0.5x speed)
       Foreground (UI/text, 1x speed)
  Effekt: 3D Raumtiefe ohne echtes 3D
  Kosten: Minimal CSS + GSAP ScrollTrigger

8. STAR ATLAS ANALYSE — CINEMATIC UNIVERSE DESIGN
================================================================================

GRUNDPRINZIP:
  Die Seite ist kein UI. Sie ist ein:
  Cinematic Interface
  = Sci-Fi HUD Interfaces + AAA Game Cinematics + Luxury Tech Branding
  Fühlt sich an wie eine Raumschiff-Konsole.

FARBSTRATEGIE (STAR ATLAS):
  Schwarz / Deep Space Black    → dominierender Hintergrund
  Blau / Electric Blue          → Technologie-Highlights
  Violett / Cosmic Purple       → Mystery, Alien, Future
  Weiß / Star Light             → Typografie, minimaler Einsatz

  Funktion:
  Schwarz   → Tiefe, Unendlichkeit, Fokus auf Licht
  Blau      → Blue = Technology, Intelligence, Trust
  Violett   → kosmische Stimmung, Unbekanntes
  Weiß      → nur für Fokus, verhindert visuelle Überladung

TIEFENILLUSION (der wichtigste technische Trick):
  Parallax Scrolling — 3 Layer:
    Stars    (langsam)  → Background
    Nebula   (mittel)   → Midground
    UI/Text  (schnell)  → Foreground
  Effekt: Nutzer fühlt sich wie in einem Raumflug

LICHTDESIGN:
  Typische Lichtquellen: Sterne, Energiekerne, Nebula Glow, Engine Trails
  Wichtige Regel: Licht kommt fast immer aus dem Bildzentrum
  → Eye Attraction Point, Blick wird automatisch dorthin gezogen

TYPOGRAFIE (STAR ATLAS STIL):
  Sans Serif, futuristisch, sehr dünne Linien
  Struktur: Huge Headline → Short Subline → Minimal Body Text
  Warum: Große Headlines kommunizieren Vision, verkaufen Größe

BILDKOMPOSITION:
  Center-Weighted Composition (Schiff / Node / Headline immer mittig)
  Augenbewegung: Bild → Objekt → Text

MOTION (STAR ATLAS):
  Slow Drift:     Sterne bewegen sich minimal → Universum wirkt lebendig
  Ship Flythrough: Objekte gleiten durch Szenen → Direction + Story
  UI Fade-ins:    Opacity + slight vertical movement → Cinematic Timing

SCROLL ARCHITEKTUR:
  Scroll = Filmschnitt (nicht Navigation)
  Jeder Scroll = Neue Szene, neue Kameraperspektive
  = Narrative Scroll Architecture

SPATIAL UI:
  Interface-Elemente sehen aus wie Radar / Navigation / Ship Dashboard
  Nutzer fühlt sich: nicht wie Besucher — sondern wie Pilot

BILDSPRACHE:
  Motive: Massive Spaceships, Planet Horizons, Space Stations, Galactic Nebula
  Skalierungstrick: Schiff = 2–5% der Szene. Space = 95–98%.
  → Universum wirkt endlos

GERÄUSCHLOSE IMMERSION:
  Vermeidet: aggressive Farben, UI clutter, laute Kontraste
  Stattdessen: Dark, Slow, Massive
  = Meditative Sci-Fi Atmosphäre

DESIGN-PHILOSOPHIE:
  "World First"
  Universe → Immersion → Information
  (nicht: Interface → Content)

ATLAS-ÜBERTRAGUNG:
  Dark Space           → Hintergrund für Atlas
  AI Nodes als Sterne  → Tools sind Nodes
  Datenströme als Linien → Workflows
  Modelle = Konstellationen → Kategorien
  Atlas Core Hub       → Navigation Center

PSYCHOLOGISCHE TRIGGER (alle gleichzeitig aktiv):

11. MOTION SYSTEM — ANIMATIONS-REGELN
================================================================================

MOTION PHILOSOPHIE:
  Slow + Smooth + Minimal = Premium
  Fast + Busy + Overloaded = Cheap

MOTION TYPEN:

  Background Drift:
    Was: Sterne / Partikel bewegen sich minimal im Hintergrund
    Speed: sehr langsam (0.02px/frame)
    Effekt: Universum wirkt lebendig, nicht statisch

  Parallax (3 Layer):
    Layer 1 (Background): 0.2× Scroll-Speed  → Stars, Nebula
    Layer 2 (Midground):  0.5× Scroll-Speed  → Particles, Lines
    Layer 3 (Foreground): 1.0× Scroll-Speed  → UI, Text
    Effekt: 3D Raumtiefe ohne echtes 3D

  Node Pulse:
    Was: AI-Nodes pulsieren sanft (scale 1.0 → 1.05 → 1.0)
    Duration: 2000–3000ms ease-in-out, infinite
    Effekt: Nodes wirken lebendig, wie Sterne

  Section Reveal:
    Was: Sections erscheinen beim Scroll (opacity 0→1, y 40→0)
    Duration: 800ms ease-out
    Trigger: "top 80%" des Viewports
    Library: GSAP ScrollTrigger

  Card Hover:
    Was: Cards heben sich leicht an (translateY -2px) + glow
    Duration: 200ms ease
    box-shadow: 0 0 20px rgba(59,130,246,0.15)

  Data Stream:
    Was: Verbindungslinien zwischen Nodes pulsen (stroke-dashoffset animation)
    Speed: 1.5s linear infinite
    Farbe: Electric Blue, opacity 0.4–0.8

  Cursor Parallax:
    Was: Kamera bewegt sich leicht mit Mausbewegung
    Strength: ±15px auf X und Y
    Effekt: Hero fühlt sich wie ein lebendiger Raum an

ANIMATIONS-STACK:
  GSAP + ScrollTrigger  → Scroll-getriggerte Szenen
  Framer Motion         → UI Micro-Interactions (Cards, Buttons, Hover)
  CSS Keyframes         → Node Pulse, Background Drift
  Three.js              → Galaxy Rotation, Node Movement

TIMING REGELN:
  Hover:      150–200ms ease
  Reveal:     600–900ms ease-out
  Scene Trans: 1000–1500ms ease-in-out
  Background: 2000–4000ms (sehr langsam)
